= Configurar e configurar AI Data Engine
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Como administrador de storage ONTAP, você usa ONTAP System Manager para integrar AI Data Engine (AIDE) ao seu ambiente AFX. Após concluir a instalação dos seus nós de computação de dados, o ONTAP pode descobri-los e adicioná-los a um cluster. Você pode adicionar nós AIDE durante a configuração inicial do cluster AFX ou adicioná-los a um cluster AFX existente e, em seguida, concluir a configuração AIDE necessária.

.Antes de começar
* Você tem NetApp DCNs ou servidores de terceiros instalados e conectados aos switches do cluster. Consulte a documentação de instalação para link:../install-aide-server/install-workflow.html["AI Data Engine software em servidores de terceiros"] ou link:../install-dcn/install-workflow.html["DCNs da NetApp"] para mais informações.
* (Para servidores de terceiros) Você concluiu link:../install-setup/install-aide-as-software.html["Instalação do software AIDE"] em cada servidor. ONTAP pode então detectar esses servidores como nós de computação, e você pode adicioná-los a um cluster.




== Configure AIDE e integre-o a um sistema de storage AFX

Existem duas maneiras de configurar AIDE e integrá-lo a um sistema de storage AFX. Selecione a opção apropriada para o seu ambiente.



=== Configurar AIDE com um novo cluster AFX

Se você estiver configurando AIDE como parte da implantação inicial de um sistema AFX, o processo de configuração do AIDE faz parte do fluxo de trabalho padrão de configuração do cluster AFX. Durante a configuração do cluster ONTAP, o System Manager descobre automaticamente os nós de computação conectados e os inclui na configuração do cluster.


NOTE: Você precisará link:../install-setup/configure-oidc.html["configurar autenticação OpenID Connect"] habilitar o acesso ao AIDE Console.



=== Configurar AIDE com um cluster AFX existente

Se você estiver adicionando nós de computação (sejam NetApp DCNs ou servidores de terceiro) a um sistema de storage AFX existente, será necessário adicionar os nós e concluir a configuração do seu cluster AIDE.



==== Adicione nós de computação ao cluster

ONTAP detecta os nós de computação conectados à sua rede de cluster e os exibe no System Manager. link:../administer/dcn-add-node.html["Adicione nós de computação ao seu cluster de armazenamento para transformá-lo em um AIDE cluster"].



==== Finalize a configuração do seu cluster AIDE

Após adicionar os nós de computação e concluir a configuração inicial da rede, você pode finalizar a configuração do seu novo cluster AIDE.

.Sobre esta tarefa
Uma janela de boas-vindas com orientações de configuração é exibida na página com os passos necessários para a configuração. Uma marca de seleção é exibida à esquerda de cada passo ou item de ação que já foi concluído. Uma seta (image:icon_arrow.gif["seta azul apontando para a direita"]) é exibida para os itens que ainda não foram concluídos. Para cada uma das ações incompletas:

* Selecione o link do título ativado e conclua a ação de configuração para o seu ambiente.
* Se o link estiver desativado, passe o cursor sobre o título para exibir a ação necessária para ativá-lo e continue com a ação de configuração.


Cada etapa possui um link para documentação adicional, quando disponível. Algumas das configurações podem estar incluídas como parte da configuração do cluster AFX em https://docs.netapp.com/us-en/ontap-afx/["Documentação da AFX"^].

.Passos
. Selecione *Configurar agregação de links e VLANs*.
+
Caso seja necessário para o seu ambiente, você pode configurar LAGs e VLANs em nível de cluster. Selecionar esta opção abrirá a página de portas Ethernet.

. Selecione *Configurar protocolos de rede*.
+
Você pode configurar os protocolos de rede da storage VM (SVM).

. Selecione *Update data compute software*.
+
Se uma imagem de software AIDE estiver disponível para atualização, esta opção será habilitada. Selecione-a para abrir a página de atualização de software. Para mais informações, consulte link:../administer/update-aide-firmware-software.html["Atualizar o software AIDE"].

. Selecione *Configurar rede do mecanismo de dados*.
+
Esta opção exibe a página para configurar o endereço de rede do AI Data Engine.

. Selecione *Criar interfaces de rede entre clusters*
+
Esta opção abre a página para configurar as interfaces de rede intercluster como parte de um procedimento de duas etapas, caso você pretenda catalogar dados remotos.

. Selecione *Interagir com outros sistemas de storage ONTAP*.
+
Após criar as interfaces de rede entre clusters, você pode estabelecer relações de peering com outros sistemas de storage ONTAP.

. Selecione *Adicionar um contêiner de dados*.
+
Você pode adicionar um volume para uso pelo AIDE e associá-lo a uma SVM específica.

. Selecione *Add Workspaces*.
+
Isso inicia a página de espaços de trabalho do Data Engine, onde você pode link:../workspaces/create-workspace.html["criar um workspace"].

. Selecione *Configurar OpenID Connect*.
+
Você precisará link:../install-setup/configure-oidc.html["configurar autenticação OpenID"] habilitar o acesso ao AIDE Console.



.Depois que você terminar
Após a conclusão de todas as ações, a janela de orientação fica oculta. Você pode reabri-la e fechá-la manualmente conforme necessário.



== Ative sua licença AIDE para implantações do NetApp DCN

Se você tiver NetApp DCNs em sua configuração, precisará link:../install-setup/install-license-task.html["Instale sua licença de serviços premium da AIDE"] após o seu cluster AIDE estar configurado e integrado ao sistema de storage AFX. A licença de serviços premium do AIDE é necessária para a funcionalidade completa do AIDE, incluindo vetorização e recursos de guardrails.
